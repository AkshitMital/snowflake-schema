# Checkpoint A Implementation Notes

## Source Identity Ambiguity

The v2 ingestion contract is clear about precedence: `SOURCE_OBJECT_ID` first, then normalized source URI, then source container plus normalized source path. The remaining ambiguity is source-specific case sensitivity.

This checkpoint preserves case for `SOURCE_OBJECT_ID` because some systems treat IDs as case-sensitive. Fallback URI/path identities are lower-cased after URL decoding and separator cleanup because markdown exports and SharePoint-style paths commonly drift in case, slash style, and transient query strings. If APS adds a future source where path case is truly meaningful, that adapter should use stable source object IDs or define a source-specific fallback policy before production load.

## CONTENT_SCOPE_KEY Ambiguity

`docs/aps_content_ingestion_contract.md` lists formulas for most deterministic keys but omits `CONTENT_SCOPE_KEY`. The v2 DDL comment says the recommended shape is `CONTENT_VERSION_KEY + scope type + entity id/name`.

This checkpoint implements:

```text
CONTENT_SCOPE_KEY =
  CONTENT_VERSION_KEY || ':scope:' || SCOPE_TYPE || ':' ||
  COALESCE(SCOPE_ENTITY_ID, NORMALIZE(SCOPE_ENTITY_NAME), NORMALIZE(SCOPE_LABEL), 'unscoped')
```

The `:scope:` literal keeps scope keys collision-resistant with other child keys. Parser implementation should confirm whether unscoped `UNKNOWN` or document-level scope rows should use `unscoped`, a controlled scope label, or a future source-specific entity identifier.

## Decisions Needed Before Parser Implementation

- Define the real APS entity dictionary sources for investments, portfolios/funds, strategies/topics, companies, and people.
- Decide whether markdown section paths should be heading slugs, positional paths, or a hybrid. Entity-link body anchors depend on this.
- Decide how to represent source bundle root paths in production manifests without leaking local machine paths into canonical lineage.
- Decide whether image OCR/caption text arrives beside markdown bundles or is generated by a later adapter stage.
- Decide the adapter versioning convention used in pilot manifests and staged records.
